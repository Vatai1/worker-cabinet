# Vacation API Implementation

## Обзор
Vacation API предоставляет RESTful интерфейс для создания заявок на отпуск через HTTP. API было создано по запросу пользователя для автоматизации процесса создания заявок.

## Структура проекта
```
/opt/data/vacation-api/
├── server.js                 # Основной сервер Express
├── package.json             # Зависимости проекта
├── routes/vacation.js       # Роуты для заявок
├── controllers/vacationController.js  # Логика обработки заявок
├── middleware/validation.js # Валидация входных данных
├── examples.sh              # Примеры использования API
├── README.md                # Документация API
└── SETUP.md                 # Инструкция по установке
```

## Основной эндпоинт
`POST /api/vacations/request` - создание заявки на отпуск

### Тело запроса
```json
{
  "employeeName": "Иванов Иван Иванович",
  "employeePosition": "Разработчик", 
  "type": "Ежегодный оплачиваемый",
  "startDate": "2025-07-01",
  "endDate": "2025-07-14",
  "year": "2025",
  "note": "Отпуск за свой счет"
}
```

### Валидация
API автоматически выполняет следующие проверки:
- Даты начала и окончания обязательны
- Даты должны быть в будущем
- Дата окончания не может быть раньше даты начала
- Для ежегодного отпуска обязательно указание года
- Поддерживаемые типы отпусков валидируются
- Количество дней вычисляется автоматически

### Пример ответа
```json
{
  "success": true,
  "message": "Заявка на отпуск успешно создана",
  "data": {
    "id": "uuid-здесь",
    "employeeName": "Иванов Иван Иванович",
    "employeePosition": "Разработчик",
    "type": "Ежегодный оплачиваемый",
    "startDate": "2025-07-01",
    "endDate": "2025-07-14", 
    "days": 14,
    "year": "2025",
    "note": "Отпуск за свой счет",
    "status": "На рассмотрении",
    "createdAt": "2024-06-03T10:00:00.000Z",
    "updatedAt": "2024-06-03T10:00:00.000Z",
    "formattedRequest": "ЗАЯВКА НА ОТПУСК\n\n..."
  }
}
```

## Запуск API
```bash
cd /opt/data/vacation-api
npm install
npm start
```

API доступно по адресу: http://localhost:3001

## Интеграция с скиллом
Скилл vacation-request теперь отправляет данные в этот API вместо ручного форматирования. API обеспечивает:
- Единообразную валидацию данных
- Автоматическое вычисление дней
- Присвоение уникальных ID
- Стандартизированный формат ответа

## Примечание
В текущей реализации данные хранятся в памяти (массив). Для production-использования необходимо добавить базу данных (например, PostgreSQL или MongoDB) и персистентное хранилище.
